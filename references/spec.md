# ESO 表格式分析 — 規則本體

理論脈絡（Elements-Structure-Order、Subject/Object/World、阻力最小路徑）不在此複製，見延伸閱讀：
https://vocus.cc/article/695600d9fd89780001c83043 （ESO 框架：一種敘事分析的方法）

本檔只放執行需要的規則本體：表格模板、精簡展演範例、三燈規則、來源注腳格式。

---

## 表格模板

```markdown
情境一句話：__________________________________________

| Sbj（是誰） | Lgc（立場與想完成的事） | Rst（什麼擋住他） | Option（接下來可能怎樣） |
| ----------- | ---------------------- | ---------------- | ----------------------- |
| 角色 A | 他是＿＿（Idt），此刻＿＿（Emt）；所以他想完成＿＿（Dbt） | 擋住他的是：□別人立場 □資源不足 □規則限制 → ＿＿ | 🟢 根據目前條件可行：＿＿ ／ 🟡 似乎可行但目前條件有缺漏：＿＿ ／ 🔴 根據目前條件不可行：＿＿（誰、為何） |
| …（每個立場方一列） | | | |

【共同條件 Wrd】所有角色都要遵守的規則：__________________________
```

橫讀一列＝一句完整的話：「這個人，因為他的身分和情緒，想完成某事；但被某原因擋住；所以接下來這樣走得通、那樣會有人不接受。」

## 精簡展演範例

**情境一句話：** 小團隊要在月底前交案，但關鍵設計師想重做整個視覺。

| Sbj | Lgc | Rst | Option |
| --- | --- | --- | --- |
| 設計師 | 他是專業把關者（Idt），此刻對現有版本感到丟臉、不甘心（Emt）；所以他想完成「整個視覺重做」（Dbt） | ☑ 規則限制 → 月底交案，重做來不及 | 🟢 局部重做關鍵幾頁（符合限制）／🟡 快速重做後提前交（需 PM 讓步）／🔴 整個重做：PM 不接受，她的 Dbt（準時交案）被踩 |
| PM | 她是進度負責人（Idt），此刻焦慮、怕失信客戶（Emt）；所以她想完成「月底準時交案」（Dbt） | ☑ 別人立場 → 設計師堅持重做 | 🟢 談出「先交堪用版、後補強」版本／🟡 爭取小幅延期（需客戶同意）／🔴 要求設計師原封不動交：設計師不接受 |

【共同條件 Wrd】所有角色都要遵守的規則：**月底這個交期是客戶合約寫死的，改不了。**

判斷示範：兩方的 Dbt 互相卡住，但共同規則把「全部重做」直接排除。所有人都能接受的路徑，只剩「先交堪用版、局部重做、下版優化」——不是最好，是唯一大家都不翻臉的。這條結論不是憑空來的，是把每列的 Dbt、Rst 和底線疊起來後，被排除法逼出來的。

## 三燈規則（Option 欄，唯一需要判斷力的欄）

- 🟢 **根據目前條件可行**：現在的條件就能走，不需要等新條件成立
- 🟡 **似乎可行但目前條件有缺漏**：方向合理，但缺了某個現在還沒到位的條件（資料、授權、驗證、時間……）
- 🔴 **根據目前條件不可行**：會讓某個具體的人（哪個 Sbj）或哪條 Wrd 不能接受——必須指名誰、為什麼，不能只寫「有風險」

**三燈都要填，不可省略任一燈。**
- 只想得出兩燈、缺 🔴 → 提示：「只想『可能』會滑成腦力激盪，逼自己想這條路誰會不接受」
- 缺 🟡 → 提示：「黃燈最常被漏想，真的沒有任何看似可行但條件不足的選項嗎？」

判斷基準是「目前條件」，不是永久判斷——條件變了，燈號可以重新評估。

## 來源注腳格式（若使用者提供具體來源時才用）

- 上標數字附在被引用的那句話**後面**
- 底部設 `## 來源` 區塊回收所有引用，放在表格最底部
- **格式優先順序（可移植性原則）**：
  1. URL 優先：`[描述文字](https://...)` — 任何人在任何系統都能開
  2. 本地檔案 wikilink 次之：`[[檔名]]`（裸檔名，不加路徑）—— 僅在使用者有對應筆記系統時有效
- 同一來源可被多處引用，共用同一序號；序號全表連續

範例：
```markdown
| A2 教師 | 他是診斷知識持有者（Idt），師生比 1:30 使個別診斷物理上不可能²（Emt）... | ☑ 規則限制 → 師生比是組織規定¹ | 🟢 引入 AI 系統...³ |

## 來源
¹ 訪談記錄 [[某檔名.md]]
² [某研究](https://...) — 說明
```
